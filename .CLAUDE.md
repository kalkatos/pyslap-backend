## Mandatory rules:

- Always read @.agents/rules/project.md for what this project is.
- Always read @.agents/rules/architecture.md for how the code is structured.
- When asked to implement a feature from the file @.agents/rules/roadmap.md, first create an "implementation plan" detailing everything that will be done as follows:
  1. A description of the feature
  2. [NEW] for each script or file that will be created (if any) and a description on what will be created
  3. [MODIFY] for each script that will be changed and a description on what to change on which method or variable, also mention which and how tests are gonna be modified
  4. [TEST] describe what and how tests will be performed to ensure quality
  5. Do not implement right away, return the "implementation plan" to the user
- The user may require changes on the implementation plan, perform them as requested
- When asked to implement what is described on the "implementation plan", follow the steps:
  1. Create a branch with the name `feature/<name_of_feature>`
  2. Implement the changes as stated on the plan
  3. Perform tests (if any) as described on the plan
  4. If all tests pass, commit the changes to git with a brief message on what changed
- Always add a space after method name declaration and before the opening parenthesis. Ignore PEP rule or any other rule that says otherwise.
```
# ✅ Correct
func my_method (param: int) -> void:
# ❌ Incorrect
func my_method(param: int) -> void:
```
- To test the game working for real, use:
  1. Run on a terminal `.venv/Scripts/python -m uvicorn local.app:app --reload --port 8000` to start the server
  2. Run two other terminal instances with `.venv/Scripts/python games/rps_client.py --port 8000 -m` to simulate two players connected
  3. You must be able to enter R P or S commands and the game must end with a Victory for one of the players.
  4. If not working as intended, perform fixes to have it functional.
  5. Terminate any terminal session created this way.
